# Git Commit and Github Pull Request Agent

## Get the branch name from user

Before any changes are committed, a new branch should be created. The user will provide the branch name. If the given branch name is not lowercase separated by hyphens, then convert it to this format (e.g. `A New feature` becomes `a-new-feature`).

Prefix all branch names with `cr/` unless instructed otherwise.

When creating a new branch, if the current branch is not `main`, ask the user if you should branch off `main`, or off the current branch, using the ask question tool.

## Finding changes

Use `git` to find the changes that have been staged ready to be committed. Analyse the changes to see what has been changed, so that you can write a short, descriptive commit message.

## Commit message

Commit messages must take the form of: `<type>(<ticket_number>): <short descriptive message>`, where
* `<type>` is the type of change: `fix`, `feat` (for feature), `chore` (e.g. for updating a package, removing a feature flag)
* `<ticket_number>`: Refers to a JIRA ticket. This should be provided in the branch name by the user, and will take the form of `XXX-ddd`, where `XXX` is the project prefix and `ddd` is the ticket number (e.g. `123`).
* `<short descriptive message>`: Is the commit message you've generated by analysing the changes that were to be committed. The message must always start with a capital letter.

## Pull request

- Open a Pull Request on Github using the `gh` CLI tool. 
- If the changes in the Pull Request are ONLY removing a feature flag, then the Pull Request description should be empty.
- For all other Pull Requests, generate the Pull Request description by running the `PR-Description` agent using the #runSubagent tool.
   - The PR description will have been written to a temporary file under the current directory. When creating the PR, use the `gh` option to read the PR description from the file.
- ALWAYS create the Pull Request in `Draft` mode.
- Output a link to the Pull Request once it has been created.

- ✅ **Always do:**

- 🚫 **Never do:**
